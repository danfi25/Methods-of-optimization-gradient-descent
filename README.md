# ЛР-1 Метод градиентного спуска

Основной язык выполнения лабораторных работ Python 3 (при желании можно экспериментировать на других языках: Julia, Mojo, С++,… но, в таком случае, нужно это предварительно согласовать). Работы выполняются индивидуально или в группах по 2 человека. Код и отчёт по лабораторной необходимо загрузить в репозиторий на GitLab (ссылка на его создание будет в таблице курса, WIP).

# Критерии оценивания

Каждое задание оценивается до 10 баллов. При этом учитывается: 

1. Работоспособность и качество кода.  
2. Полнота отчета.  
3. Знание теории, которая лежит в основе применяемых методов.  
4. За особенно полное и интересное исследование баллы могут быть увеличены.

# Базовое задание (10 баллов)

1. Изучите коллекцию классических функций, интересных для задач минимизации [Benchmark functions | BenchmarkFcns](https://benchmarkfcns.info/fcns). Выберите не менее 3 функций двух переменных для экспериментов. Найдите точное значение минимумов.

2. Найдите минимум функции в разных начальных точках, используя библиотечную реализацию scipy.optimize.minimize с методами первого порядка.

3. Реализуйте метод градиентного спуска с различными стратегиями выбора шага (learning rate scheduling) и выхода из цикла. Найдите значение минимума у выбранных функций. Обратите внимание на используемые гиперпараметры в методах. В реализации их следует выносить в интерфейс метода.

4. Сравните результаты экспериментов на выбранных функциях. На них могут влиять: выбор начальной точки и критерий остановки, выбор шага, значения гиперпараметров, число обусловленности функции. Эффективность метода описывается зависимостью точности, количества итераций, количества вычислений значений минимизируемой функции и её производной (градиента).

# Расширенное задание (10 баллов)

* **1-extended.** Используйте для экспериментов функции с более сложным поведением. Это могут быть функции нескольких переменных, функции без точного аналитического ответа для задачи минимизации, функции с зашумленными значениями. Можно рассмотреть параметрическое семейство функций, то есть задать зависимость от параметра. При изменении параметра может меняться, например, степень мультимодальности или зашумлённости.

* **3-extended.** Реализуйте метод градиентного спуска с линейным поиском величины шага. Линейный поиск можно осуществлять любыми методами. 

# Требования к оформлению отчёта

В описании метода укажите, это реализованный Вами или библиотечный метод, из какой библиотеки он взят, или какую библиотеку использует внутри реализации, если использует. При наличии, указывайте важные особенности Вашей реализации метода, они могут быть алгоритмические (например,  имеющиеся гиперпараметры), так и технические (например, использование мемоизации в алгоритме).

Нарисуйте графики с линиями уровня и траекториями методов (можно в области задания, 2D; можно на графике функции в 3D).

Результаты исследования сопровождаются графиками зависимости исследуемых показателей и таблицами. На их основе надо сделать выводы о применимости и эффективности тех или иных методов.

В случае выполнения работы на Python отчёт рекомендуется делать в Jupyter Notebook. При этом реализация алгоритмов должна быть вынесена в отдельные модули и импортироваться в ноутбуке. При выполнении работы на другом языке отчёт можно делать в PDF, Markdown и т.п.
